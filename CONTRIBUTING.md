# Contributing

## Local Setup

```bash
pnpm install
pnpm dev
pnpm cf-typegen
```

This repo expects:

- Node `24.x`
- pnpm `10.28.2`

Those versions are declared in [`package.json`](./package.json).

## Before Opening A PR

Run the same checks CI runs:

```bash
pnpm check
pnpm typecheck
pnpm test
pnpm coverage
pnpm build
pnpm test:e2e
```

If you want the shorter local baseline first, start with:

```bash
pnpm check
pnpm typecheck
pnpm test
pnpm build
```

## Testing Notes

- Unit tests use Vitest with [`vitest.config.ts`](./vitest.config.ts).
- E2E tests use Playwright with [`playwright.config.ts`](./playwright.config.ts).
- Playwright starts Vite directly instead of `pnpm dev` so the dev server shuts down cleanly on Windows.
- If you add or change Cloudflare bindings, rerun `pnpm cf-typegen` before opening a PR.

## Formatting And Linting

- Biome is the formatter and linter for this repo.
- Run `pnpm format` to apply formatting changes.
- Run `pnpm check` to run Biome checks without writing changes.
- Tailwind CSS v4 directives are enabled in [`biome.json`](./biome.json), so keep that config in place unless the styling stack changes.

## Generated Files

- [`src/routeTree.gen.ts`](./src/routeTree.gen.ts) is generated by TanStack Router.
- [`worker-configuration.d.ts`](./worker-configuration.d.ts) is generated by Wrangler.
- Do not hand-edit generated route tree output.

## Deployment Notes

- Cloudflare Workers configuration lives in [`wrangler.jsonc`](./wrangler.jsonc).
- `pnpm preview` is the closest local deployment check before publishing to Cloudflare.
- `pnpm deploy` builds the app and hands deployment to Wrangler.

## PR Guidance

- Prefer small PRs with a single focused change.
- If you touch routing, add or update at least one relevant test.
- If you change shared tooling or config, update [`README.md`](./README.md) and this file when the workflow changes.

# Project Kelsier

Open-source exploration of cognition, communication patterns, and human-centered collaboration.

[[Live Demo](https://project-kelsier.mindphreak.workers.dev/)] | [Docs] | [Roadmap]

Built with TanStack Start, React, Vite, Biome, Vitest, Playwright, GitHub Actions CI, and Cloudflare Workers-ready deployment.

## Commands

```bash
pnpm install
pnpm dev
pnpm check
pnpm typecheck
pnpm test
pnpm coverage
pnpm build
pnpm preview
pnpm cf-typegen
pnpm deploy
pnpm test:e2e
```

## Tooling Notes

- Cloudflare Workers deployment is configured through [`wrangler.jsonc`](./wrangler.jsonc) and the Cloudflare Vite plugin in [`vite.config.ts`](./vite.config.ts).
- [`worker-configuration.d.ts`](./worker-configuration.d.ts) is generated by Wrangler and should be refreshed with `pnpm cf-typegen` after adding or changing Cloudflare bindings.
- Vitest uses [`vitest.config.ts`](./vitest.config.ts) instead of sharing [`vite.config.ts`](./vite.config.ts). This avoids the full TanStack Start and Cloudflare runtime plugin stack keeping the unit test process open.
- Playwright starts Vite directly in [`playwright.config.ts`](./playwright.config.ts) instead of using `pnpm dev`. This is a Windows-friendly workaround so the dev server shuts down cleanly after E2E runs.
- Biome has Tailwind CSS directive parsing enabled in [`biome.json`](./biome.json) because Tailwind v4 syntax like `@theme` and `@plugin` is used in [`src/styles.css`](./src/styles.css).
- [`src/routeTree.gen.ts`](./src/routeTree.gen.ts) is generated by TanStack Router and is intentionally excluded from the normal editing and linting flow.

## Cloudflare Workers

This repo is configured for Cloudflare Workers rather than static-only hosting.

Typical deploy flow:

```bash
pnpm install
pnpm cf-typegen
pnpm build
pnpm preview
pnpm deploy
```

If you add Cloudflare bindings later, update [`wrangler.jsonc`](./wrangler.jsonc) and rerun `pnpm cf-typegen`.

## CI

GitHub Actions runs the baseline checks on pushes to `main` and on pull requests:

- `pnpm check`
- `pnpm typecheck`
- `pnpm test`
- `pnpm coverage`
- `pnpm build`
- `pnpm test:e2e`

## Tooling Notes

- Vitest uses [`vitest.config.ts`](./vitest.config.ts) instead of sharing [`vite.config.ts`](./vite.config.ts). This avoids long-lived app plugins such as Nitro, TanStack Start, and devtools keeping the unit test process open.
- Playwright starts Vite directly in [`playwright.config.ts`](./playwright.config.ts) instead of using `pnpm dev`. This is a Windows-friendly workaround so the dev server shuts down cleanly after E2E runs.
- Biome has Tailwind CSS directive parsing enabled in [`biome.json`](./biome.json) because Tailwind v4 syntax like `@theme` and `@plugin` is used in [`src/styles.css`](./src/styles.css).
- [`src/routeTree.gen.ts`](./src/routeTree.gen.ts) is generated by TanStack Router and is intentionally excluded from the normal editing and linting flow.
- [`src/routes/__root.tsx`](./src/routes/__root.tsx) contains a targeted Biome ignore for the inline theme bootstrap script. That script runs before hydration to prevent a flash of the wrong theme.

## CI

GitHub Actions runs the baseline checks on pushes to `main` and on pull requests:

- `pnpm check`
- `pnpm typecheck`
- `pnpm test`
- `pnpm coverage`
- `pnpm build`
- `pnpm test:e2e`
